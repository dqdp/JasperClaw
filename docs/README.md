# Docs

Purpose:
This directory is the canonical documentation layer for architecture, contracts, planning, and operations.

Start here:
- `implementation-status.md`: open first when you need to separate implemented behavior from accepted design.

Index:
- `architecture.md`: open when you need the global v1 system shape and invariants.
- `implementation-status.md`: open when checking what is real versus placeholder or planned.
- `roadmap.md`: open when deciding milestone order and delivery sequencing.
- `backlog.md`: open when translating milestones into executable epics and tasks.
- `data-model.md`: open when touching persistence, transcript state, audit state, or memory schema.
- `handoff_to_agent.md`: open when you need the compact implementation brief for this repo.
- `adr/`: open when a change touches an accepted architectural decision.
- `features/`: open when implementing feature-specific runtime behavior.
- `ops/`: open when changing configuration, observability, error semantics, or action policy.
- `runbooks/`: open when performing operational procedures rather than changing design.
- `service-contracts/`: open when implementing or consuming canonical service interfaces.
- `testing/`: open when you need the short-form TDD planning template.
